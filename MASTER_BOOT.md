## What is this?

**The Guild of Builders** is a story-driven, open-source series that teaches children
Python, software engineering, AI, and computational thinking by building **one evolving
game across four volumes** вАФ *Captain Byte's Treasure Quest*.

We are not teaching Python.
We are raising **Builders**.

## Boot sequence (read in this order the first time)

1. `docs/AI_WORKING_AGREEMENT.md` вАФ **how we work.** Read this first, every time.
2. `docs/PROJECT_BIBLE.md` вАФ understand *why* before *what* (includes the Widening Circle).
3. `docs/DECISIONS.md` вАФ what's already settled (LOCKED). Don't reopen it.
4. `docs/CHARACTERS.md` вАФ meet the crew (and the Fog Creatures).
5. `docs/STYLE_GUIDE.md` вАФ learn the voice and the accessibility rules.
6. `docs/ROADMAP.md` вАФ see where we're sailing next.
7. `docs/SERIES_OUTLINE.md` + `docs/GAME_DESIGN.md` вАФ the chapter map and what we're building.
8. For deeper context when relevant: `docs/CURRICULUM.md`, `docs/BUILDERS_LOGBOOK.md`,
   `docs/SETUP.md`, and (for Volume IV) `docs/HUMAN_AI_MANIFESTO.md`.
9. Then open the volume you're working on in `manuscript/`, `workbook/`, `source_code/`.

## The two rules that keep this project alive

**Rule 1 вАФ One artifact at a time.**
We don't rewrite everything in a chat. We edit **one file**, ship a new version, commit it.

**Rule 2 вАФ The North Star.**
Before adding anything, ask:

> **"Will this help a child become a better Builder?"**

If yes, it belongs.
If no, it waits in the Treasure Chest for another adventure.

## The Donut Law рЯН©

Every working session ends with a **real deliverable** вАФ a file you can download, read, or
run. Never a promise. Never a "Mommy's Donut." An actual donut, committed to the repo.

If a session ends and there's no donut, the session isn't finished.
