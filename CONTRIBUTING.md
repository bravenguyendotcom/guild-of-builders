# 🤝 CONTRIBUTING to the Guild of Builders

Welcome, Builder. Here's how to add to this project without breaking its spirit.

## The one test that governs everything

Before you add anything — a chapter, a mission, a feature, a character, a joke — ask:

> **"Will this help a child become a better Builder?"**

If yes, it belongs. If no, it waits in the Treasure Chest. That's the North Star.

## The workflow

1. **One artifact at a time.** Work on a single file per session. Don't rewrite the world.
2. **Read before you write.** Check `docs/PROJECT_BIBLE.md`, `docs/CHARACTERS.md`, and
   `docs/STYLE_GUIDE.md` so your addition stays consistent.
3. **Ship a version.** When a document meaningfully changes, note it (v1.0 → v1.1).
4. **Deliver a donut.** 🍩 Every session ends with a real, committed file. No promises.
5. **Update the roadmap.** Add a row to the session log in `docs/ROADMAP.md`.

## Git commit messages (keep them human)

Use short, clear messages:

```
docs: lock the four-volume roadmap
manuscript: add Volume I, Chapter 0 (Welcome)
workbook: draft Mission 1 (Hungry Pizza Robot)
game: v1 treasure chest opens with guess-the-number
fix: Dragon Debug's tea line was off-voice
```

## Respecting the characters

Characters must behave consistently across all volumes (`docs/CHARACTERS.md`). If a line
doesn't sound like something that character would say, don't write it.

## Accessibility is not optional

Teppy has mild dyslexia. Short lines, generous whitespace, plain words, no walls of text.
See `docs/STYLE_GUIDE.md`.

## Decisions

If you make a real decision (something future contributors shouldn't reopen), record it in
`docs/DECISIONS.md` as LOCKED or OPEN, with the reason.

---

🐉 *Make it work. Make it clear. Make it kind.*
