# ✅ workbook/ — retired as a separate track (see DECISIONS.md D-26)

> **Status:** RETIRED as an authored track. Kept as a signpost, not a workspace.

## Where the missions went

Missions are **not** written here anymore. As of **`DECISIONS.md` D-26**, every mission lives
**inside the story chapter** in `manuscript/` — one flowing read that folds the story, the
coding task, the honest teaching-bug, and the achievement together.

Why: it's kinder to read (better for Teppy — no story-then-worksheet gear-change), it roughly
halves the authoring work, and it matches how the shipped chapters (0–7) are already written.

## What to use instead

- **A mission to *do*?** → open the matching chapter in **`manuscript/volume_1/`**.
  The worksheet habits we kept — the **Testing Table** (`Input | Expected | Actual`) and the
  **Debug Log** — now live as **boxes inside the chapter**, not on a separate sheet.
  (Their rules are in **`docs/STYLE_GUIDE.md`**.)
- **Pure typing / practice?** → that's a *different* track: the **Typing Dojo**
  (**`typing_dojo/`**), with its own one-screen format (no worksheets — `DECISIONS.md` D-22).

## About `_TEMPLATE_mission.md`

worksheet-heavy 12-part shape it describes was replaced by the embedded-in-chapter model (D-26).

## If you're an author or AI booting in

points at `workbook/` as an active track, it's stale — **`DECISIONS.md` D-26 is the source of truth.**
